### **v1.1 - 风险平价优化（波动率倒数加权）**

**配置参数**：
- 选股逻辑：与 v1.0 完全相同
- 权重方案：波动率倒数加权（Inverse Volatility Weighting）
  - 平滑项：`epsilon = 1e-6` 防止除零
  - 归一化：权重和严格等于 1.0
- 调仓频率：每月第一个交易日
- 交易成本：单边 0.1%

**绩效表现**：

| 指标 | 数值 | vs v1.0 变化 | 说明 |
|------|------|--------------|------|
| **总收益率** | 435.15% | ⬇️ -104.46% | 收益下降明显 |
| **年化收益率** | 11.84% | ⬇️ -1.34% | |
| **年化波动率** | 17.93% | ⬇️ -0.22% | 波动率只下降了一点点 |
| **夏普比率** | 0.66 | ⬇️ -0.07 | 反而下降了！ |
| **卡玛比率** | 0.29 | ⬇️ -0.04 | |
| **最大回撤** | -40.62% | ⬆️ -0.11% | 回撤反而略增 |
| **日胜率** | 51.82% | ⬇️ -0.24% | |
| **月胜率** | 55.80% | ─ 0.00% | |
| **年化换手率** | 1491.15% | ⬆️ +87.80% | 换手率增加 |
| **累计交易成本** | 20.75% | ⬆️ +1.22% | 交易成本上升 |

**优化逻辑**：
- 低波动率股票获得更高权重，降低组合整体风险
- ❌ **实际结果与预期相反**：波动率只下降了 0.22%，但收益率下降了 1.34%

**重要发现**：
> 📌 **这个优化是负向的！** 原因分析：
> 1. 我们已经选出了波动率最低的 20 只股票，它们之间的波动率差异不大
> 2. 在这个范围内，等权重下"稍高波动但动量强"的股票贡献了更多收益
> 3. 波动率倒数加权过度压低了高波动（但高动量）股票的权重
> 4. 换手率反而上升，交易成本增加了 1.22%

**结论**：**v1.1 不推荐使用，回退到 v1.0 等权重**

### **v1.2 - 缓冲带机制优化（大幅降低换手率）**

**配置参数**：
- 选股逻辑：动量前50 → 波动率排序 → 缓冲带优先保留老仓位
- 核心优化：引入 `buffer_keep_n=40` 缓冲带机制
  - 上一期持仓只要在「动量前50」且「波动率前40」就优先保留
  - 不足20只时按波动率排名填补空缺
- 权重方案：等权重（Equal Weight）
- 调仓频率：每月第一个交易日
- 交易成本：单边 0.1%

**绩效表现**：

| 指标 | 数值 | vs v1.0 变化 | 说明 |
|------|------|--------------|------|
| **总收益率** | 695.66% | ⬆️ +156.05% | 收益提升显著 |
| **年化收益率** | 14.84% | ⬆️ +1.66% | |
| **年化波动率** | 18.44% | ⬆️ +0.29% | 波动率微增可接受 |
| **夏普比率** | 0.80 | ⬆️ +0.07 | 风险调整后收益改善 |
| **卡玛比率** | 0.39 | ⬆️ +0.06 | |
| **最大回撤** | -38.47% | 🔼 -2.04% | 回撤收窄 |
| **日胜率** | 51.74% | ⬇️ -0.32% | |
| **月胜率** | 56.91% | ⬆️ +1.11% | |
| **年化换手率** | 814.85% | 🔽 **-588.50%** | **下降 42%！** |
| **累计交易成本** | 11.34% | 🔽 **-8.19%** | **节省 8.2% 交易成本！** |

**优化逻辑**：
- 解决「排名边缘频繁切换」导致的无谓换仓问题
- 老仓位只要还在合格范围内就继续持有，避免「第20名踢掉第21名」这种无意义的买卖

**关键发现**：
> 📌 **这是一个非常成功的优化！核心指标全部改善：**
> 1. 年化换手率从 1403% 降至 815%，**下降 42%**，累计交易成本减少 8.2%
> 2. 由于交易成本大幅降低，**年化收益反而提升 1.66%**
> 3. 夏普比率从 0.73 → 0.80，最大回撤也收窄了 2%
> 4. 策略本质没变（还是动量+波动率选股），只是降低了摩擦成本

**结论**：**v1.2 是当前最佳版本，推荐使用**

## 📝 研究笔记与观察

### 2026-04-22 - 初始研究笔记
- **关于换手率**：当前策略每次几乎 100% 换仓，这符合动量策略的特性（动量股轮换快），但交易成本是重要优化方向
- **关于回撤**：-40.51% 的最大回撤发生在什么时间段？值得分析是否与特定市场环境相关
- **下一步建议**：先完成 v1.1 的回测，验证波动率倒数加权的效果

### 2026-04-22 - v1.1 测试结论
- **重要发现**：波动率倒数加权 **反而降低了策略表现**
- **原因分析**：
  1. 我们已经选出了波动率最低的 20 只股票，它们之间的波动率差异很小
  2. 在这个样本内，"稍高波动但动量更强"的股票实际上贡献了更多收益
  3. 波动率倒数加权过度压低了这些股票的权重，导致收益下降幅度超过了波动率下降幅度
- **启示**：在已经做过波动率筛选的前提下，权重优化的边际效益很低，甚至可能为负
- **下一步建议**：转向其他优化方向，比如：
  - 动量/波动率综合排名（不是两步筛选，而是直接排序）
  - 降低调仓频率（减少交易成本 20% → 年化收益直接 +2%）
  - 大盘趋势过滤

### 2026-04-25 - v1.2 缓冲带机制大获成功
- **核心问题**：v1.0 每次调仓换手率接近 100%，很多是「排名边缘无谓切换」导致的摩擦成本
- **解决方案**：引入 `buffer_keep_n=40 缓冲带机制
  - 老仓位只要在动量前50且波动率前40就优先保留
  - 不足20只时才按排名填补空缺
- **结果超出预期**：
  - 年化换手率从 1403% 降至 815%，**下降 42%**
  - 累计交易成本从 19.53% 降至 11.34%，节省了 8.2%
  - 由于摩擦成本降低，**年化收益反而提升 1.66%**
  - 夏普比率从 0.73 → 0.80，最大回撤也收窄了 2%
- **重要启示**：在量化策略优化中，「降低摩擦成本」往往比「追求阿尔法」更直接有效
- **下一步建议**：继续探索其他降低换手率/交易成本的方向，比如：
  - 降低调仓频率（季度调仓）
  - 动量/波动率综合排名
  - 大盘趋势过滤
